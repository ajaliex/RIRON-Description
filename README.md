# 税理士試験（法人税法）理論対策Webアプリ

本リポジトリは、税理士試験（法人税法）の理論問題対策をサポートするための学習ノート・アプリケーションです。
フロントエンドをGitHub Pagesで配信し、バックエンドのAPIサーバーとしてGoogle Apps Script（GAS）を利用しています。生成AI（Gemini API）を活用して、論点に基づいた問題生成と、入力された回答に対するフィードバックを行います。

## 全体構成
* **フロントエンド (`docs/`)**: HTML/CSS/JavaScript (GitHub Pagesで公開)
* **バックエンド (`gas/`)**: Google Apps Script (Webアプリとしてデプロイ)
* **生成AI**: Gemini 3.0 Flash

## 前提条件
1. Googleアカウント
2. GitHubアカウント
3. Gemini APIキー (Google AI Studioから取得)
4. (任意) Node.js および `clasp` (GASデプロイ用)

## セットアップ手順

### Step 1. リポジトリをクローン
当リポジトリをご自身の環境にクローン（またはフォーク）してください。

### Step 2. APIキーの設定 (GAS スクリプトプロパティ)
コード内にAPIキーを直接書き込むのはセキュリティ上危険です。以下の手順でGoogle Apps Scriptの**スクリプトプロパティ**にAPIキーを安全に保存します。

1. 後述の「Step 4」でGASプロジェクトを作成し、ファイルをアップロードした後に行います。
2. GASエディタの左側メニューから **「プロジェクトの設定」（歯車アイコン）** をクリックします。
3. 一番下までスクロールし、「スクリプト プロパティ」のセクションで **「スクリプト プロパティを追加」** をクリックします。
4. プロパティに `GEMINI_API_KEY` と入力し、値に取得したAPIキーを貼り付けます。
5. 「スクリプト プロパティを保存」をクリックします。
   *(※これにより、コード上にキーを直接書かずに安全にAPIを呼び出すことができます。)*

### Step 3. 論点HTMLファイルの追加
`tools/add_topic.bat` を使って、学習内容となるHTMLファイルを追加します。
バッチファイルに追加したいHTMLファイルをドラッグ＆ドロップするか、コマンドラインで引数として渡すと、自動で `gas/data/` フォルダへコピーされインデックスが更新されます。

### Step 4. GASプロジェクトの作成とデプロイ

Google Apps Script (GAS) にプログラムをアップロードするには、**【方法A】コマンドラインツール(clasp)を使う方法（推奨）** か、**【方法B】ブラウザから手動でコピペする方法** の2通りがあります。

#### 【方法A】コマンドラインツール (clasp) を使う方法（推奨・一括アップロード）
複数の論点（HTML）を扱う場合、手動でのアップロードは大変なため、Google公式ツールの `clasp` の使用を強くお勧めします。

0. **【重要】** 初めてclaspを利用する場合、事前に [Google Apps Script API 設定ページ](https://script.google.com/home/usersettings) にアクセスし、**「Google Apps Script API」を「オン」** にしてください。
1. Node.js がインストールされていることを確認します。
2. コマンドプロンプト等を開き、`clasp` をインストール＆ログインします。
   ```bash
   npm install -g @google/clasp
   clasp login
   ```
3. `gas` フォルダに移動し、新しいGASプロジェクトを作成します。
   ```bash
   cd gas
   clasp create --title "税理士試験・理論対策アプリ"
   ```
4. このままプッシュ（アップロード）を実行します。フォルダ内のファイルが一括でGASに反映されます。
   ```bash
   clasp push
   ```
5. ターミナルに表示されたURL、または `clasp open-script` コマンドでGASのエディタ画面を開きます。

#### 【方法B】ブラウザから手動で作成する方法
1. Google Apps Scriptのダッシュボード（[script.google.com](https://script.google.com/)）を開き、「新しいプロジェクト」を作成します。
2. プロジェクトの設定（左側の歯車アイコン）から「**「appsscript.json」マニフェスト ファイルをエディタで表示する**」にチェックを入れます。
3. エディタ左側の「＋（ファイルを追加）」ボタンをクリックし、ファイルを1つずつ作成して中身をコピー＆ペーストします。
   * **スクリプトの場合 (`.gs`)**: 「スクリプト」を選び、`main`, `gemini`, `parser`, `config`, `data_index` という名前で作成し、各ファイルの中身をペースト。
   * **マニフェストの場合 (`appsscript.json`)**: 最初からある `appsscript.json` に上書きペースト。
   * **論点データ（HTML）の場合**: 「HTML」を選び、**ファイル名に `data/1-3_納税地`** と入力して作成し、HTMLの中身をペースト。（※GASではフォルダが作れないため、名前に `data/` を含める疑似フォルダ構成にします。以降、論点を追加するたびにこの作業が必要です。）

#### ウェブアプリとしての公開（デプロイ）
ファイルの準備・アップロードができたら、いよいよアプリを公開します。
1. 先ほど設定した「Step 2」の**スクリプトプロパティ（APIキー）** が登録されているか、念のため確認します。
2. GASエディタ右上の青い「**デプロイ**」ボタン >「**新しいデプロイ**」を選択します。
3. 種類の横にある歯車マークから「**ウェブアプリ**」を選択します。
4. 以下の設定になっていることを確認し、「デプロイ」をクリックします。
   * 次のユーザーとして実行: **「自分」**
   * アクセスできるユーザー: **「全員」**
5. デプロイ完了時に表示される **「ウェブアプリ」のURLをコピー** します。

### Step 5. フロントエンドの設定
`docs/app.js` を開き、1行目付近のURL設定部分を先ほどコピーしたURLに書き換えます。
```javascript
const GAS_API_URL = 'YOUR_GAS_WEBAPP_URL_HERE'; // ★コピーしたURLを貼り付ける
```

### Step 6. GitHub Pagesの公開
1. 変更をすべてコミットし、ご自身のGitHubリポジトリへプッシュします。
2. リポジトリの [Settings] > [Pages] を開きます。
3. Sourceを「Deploy from a branch」にし、対象ブランチを選択して配信フォルダを `/docs` に設定します。
4. Saveボタンを押すと数分でサイトが公開されます。公開されたURLにアクセスして動作を確認してください。

## トラブルシューティング
* **「論点の取得に失敗しました」と表示される**
  * `docs/app.js` に設定した `GAS_API_URL` が正しいか確認してください。
  * GASのデプロイ時のアクセス権限が「全員」になっているか確認してください。
* **「問題の取得に失敗しました」「評価の取得に失敗しました」と表示される**
  * Gemini APIキーが正しく設定されているか確認してください。
  * GAS側でエラーが起きている可能性があります。GASダッシュボードの「実行数」または「エラーログ」を確認してください。

## 重要な注意事項
* Google Apps Scriptの実行時間（6分）の制限にはご注意ください。極端に長い論点テキストを送信するとAPIがタイムアウトする可能性があります。
* APIキーはGAS側にのみ保存され、フロントエンドからは直接参照されない仕組みになっているため安全です。
